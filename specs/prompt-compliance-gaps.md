## Vision

Close the gaps between the protocol compliance reference (`~/.claude/docs/protocol-compliance-reference.md`) and the system prompt text generated by the Rust harness. Every rule the reference defines for the coding agent must be present in the prompt injected at session start.

## Requirements

1. All items from "Working Standards" in the reference must appear in the system prompt.
2. All items from "Prohibitions" in the reference must appear in the system prompt.
3. All Tier 2 Principles that are marked as injected at task-start time must be represented in the system prompt.
4. The change must propagate to all prompt consumers in the codebase (or all consumers must be documented as deriving from the central source).
5. All existing tests must continue to pass.

## Acceptance Criteria

- [x] Working Standard: IDs must use `crypto.randomUUID()` / `uuid.uuid4()`, never `Date.now()` or `Math.random()`
- [x] Working Standard: Output files → `output/`, logs → `logs/`, never bare at project root
- [x] Working Standard: JS/TS toolchain (ESLint + strict + Prettier, 80-char, semicolons, single quotes)
- [x] Working Standard: Python toolchain (Black + Ruff + mypy strict, 88-char, snake_case/PascalCase)
- [x] Prohibition: Never edit `~/.claude/settings.json` without explicit plan approval
- [x] Prohibition: Never implement backend code directly in DeepSeek mode when task touches 5+ files
- [x] Prohibition: Never trust delegated output without reading every modified file and re-running tests
- [x] Pre-Execution Reasoning section injected (problem restatement, scope boundary, options, assumptions, minimum viable change, pre-mortem)
- [x] All prompt consumers use `load_system_prompt()` from `runtime::prompt` — no duplicate/divergent prompt sources
- [x] `cargo test --workspace` — 0 failures
- [x] `cargo clippy --workspace --all-targets` (without `-D warnings`) — exit 0

## Technical Decisions

All changes are confined to `rust/crates/runtime/src/prompt.rs`, specifically:
- `get_simple_doing_tasks_section()` — extended with 7 new bullets
- `get_pre_execution_reasoning_section()` — new function
- `SystemPromptBuilder::build()` — new section wired in

This is the central source; all callers (`tools/src/lib.rs`, `rusty-claude-cli/src/main.rs`) call `load_system_prompt()` which invokes `SystemPromptBuilder::build()`. Changes propagate automatically.
